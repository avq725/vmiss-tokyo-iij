## 三、东京 IIJ 线路架构解析：三网怎么走，绕不绕路？

聊 IIJ VPS 不聊线路就是耍流氓。先把这条线路的回程和去程拆开看。

**回程（机房 → 国内）**：这是用户体验最直接的部分。东京 IIJ 系列的回程三网基本都走 IIJ 对接国内骨干直连——

- 电信回程：从东京 IIJ 出来，经 203.215.x.x（电信日本节点）→ 202.97.x.x（电信 163 骨干）直连回国，到广州、上海、北京的延迟大致在 54–80 ms 区间；
- 联通回程：经 IIJ → 219.158.x.x（联通 4837 骨干，部分走 9929）直连回国，到上海联通 80 ms 出头；
- 移动回程：经 IIJ → 223.120.x.x（CMI）→ 221.183.x.x（移动 9808）回国，到上海移动 55 ms 左右，是三网里最快的。

**去程（国内 → 机房）**：去程三网路径略有差异——

- 电信去程：走 CN2 → 香港 → NTT → 东京，有部分段会绕香港 NTT 节点；
- 联通去程：走 9929 → NTT 直连东京；
- 移动去程：走 CMI → 香港 → NTT → 东京。

简单说，**回程是这条线最大的卖点**，三网全程 IIJ/CMI 直连，没绕美西、没绕欧洲。去程因为有 NTT 中转，偶尔会有抖动，但整体算稳。这也是为什么大家搜「东京 IIJ VPS 测评」时会反复看到「三网回程直连」「不绕路」这些关键词。

## 四、性能实测：CPU、内存、磁盘 IO 跑分

光看线路不够，硬件底子也得摸一摸。综合多份第三方测评数据，东京 IIJ 系列的硬件表现大致如下：

**不适合的场景**：

- **大流量常驻**：流量是月配额，4TB 看着多，常驻高带宽场景跑超了会限速或扣费。
